# WOL 網頁內容提取工具 (WOL Scraper)

這是一個專門給 [wol.jw.org](https://wol.jw.org) 用的「**整頁目錄 → 批次抓文章**」小工具。  
重點不是單篇手動複製，而是：

- 對「守望臺線上書庫目錄」：自動展開**最多兩層**目錄，把底下所有文章抓出來
- 對「一般書籍、單一目錄」：掃一層就能把那一頁所有文章列出來
- 讓你勾選要的篇章，最後一次提取 + 匯出 TXT / HTML

---

## 支援的目錄類型與深度

工具會根據你貼上的網址，自動判斷要走幾層：

### 1. 守望臺類型（兩層目錄）

例如「洞悉」、「守望臺」、「警醒！」等，常見是這種結構：

- 第 0 層：某個年度或分類目錄頁  
- 第 1 層：期數、章節、小目錄（`/wol/toc/r24/lp-ch/...` 或 `/wol/library/r24/lp-ch/...`）
- 第 2 層：實際文章頁（`/wol/d/r24/lp-ch/...`）

工具流程：

1. 先在你給的頁面上找所有文章連結：  
   - 連結包含 `/wol/d/r24/lp-ch/` → 直接視為文章  
   - 這種情況通常是一層目錄就有完整文章列表（例如某卷的所有章節）
2. 如果找不到任何文章連結，才會啟動「**兩層展開模式**」：
   - 先收集所有子目錄連結：
     - `/wol/toc/r24/lp-ch/...`
     - `/wol/library/r24/lp-ch/...`
   - 逐一打開每個子目錄頁，從裡面找 `/wol/d/r24/lp-ch/...` 文章
   - 若子目錄裡仍然只有更多目錄（第三層），會再往下展開一次，才能把真正的文章抓出來

簡單來說：  
**守望臺型目錄 → 先掃當層文章，沒有就自動展開兩層，把底下所有文章全部列出來給你勾**

### 2. 一般書籍 / 單一目錄（單層）

例如某本書或某一冊資料，只是單純一個「文章列表頁」：

- 頁面上直接有很多 `/wol/d/r24/lp-ch/...` 連結
- 工具會一次把這些文章抓出來，**不會多跑子目錄**

這種情況性能最好，因為只打一個頁面 + n 個文章，不會多跑額外目錄頁。

## 使用教學（一步一步）

### 第一步：開啟工具頁

1. 下載這個專案或透過 GitHub Pages 開啟 `index.html`。
2. 打開瀏覽器 → 用檔案或網址開啟工具畫面。

畫面上會看到三個主區塊：

- 上方：目標目錄網址 + 自訂檔案名稱
- 中間：按鈕流程 + 系統狀態條
- 下方：連結列表（勾選用）與提取預覽

### 第二步：填入目標網址

在「**目標目錄網址**」輸入 WOL 的某個目錄頁，例如：

- 守望臺、洞悉這種「會再展開子目錄」的頁面
- 某本書或某年期的文章列表頁

工具不限制一定是哪一種，只要是 wol.jw.org 的頁面，裡面有 `/wol/d/r24/lp-ch/` 或 `/wol/toc/...` / `/wol/library/...` 的結構，就能運作。

同時可在右邊「**自訂檔案名稱與大標題**」欄位輸入這次匯出的標題，例如：

- `洞悉卷一_2026-03彙整`
- `守望臺_2019年_生活應用相關文章`

這個名字會用在：
- TXT 檔案檔名
- HTML 檔案檔名
- HTML 頁面的大標題 `<h1>`

### 第三步：按「1. 獲取頁面連結」

按下這顆按鈕後，工具會依網址類型做不同處理：

1. **先抓你貼的那一頁 HTML**
2. 優先找 `/wol/d/r24/lp-ch/` 文章連結
   - 找到 → 當作「單層目錄」（一般書籍 / 單層守望臺列表），直接把這些文章列在下方
3. 全頁找不到 `/wol/d/...` 文章時：
   - 視為「守望臺多層目錄」
   - 抓出 `/wol/toc/r24/lp-ch/...`、`/wol/library/r24/lp-ch/...` 子目錄
   - 一個一個打開子目錄頁，再從裡面抓文章
   - 遇到只有子目錄沒有文章時，會再展開「第三層」子目錄一次，去抓其中的文章

系統狀態區會顯示目前進度，例如：

- `第一層找到 12 個子目錄，正在展開第二層...`
- `展開第二層子目錄 (3/12)：某某期守望臺`

處理完之後，下方「連結列表區」會出現：

- 每個文章的標題與勾選框
- 左邊有編號（1、2、3…），方便用範圍操作

此時預設會**全部勾選**。

### 第四步：用勾選 / 篩選 / 範圍挑文章

這一段是這個工具的重點操作區。

你可以這樣用：

1. **全選 / 全不選**
   - 列表上方有一個主勾選框：
     - 勾 → 全部選取
     - 取消 → 全部取消
   - 當部分有勾，主勾選框會是「虛線狀態」（瀏覽器的 indeterminate）

2. **依關鍵字篩選**
   - 右上角「關鍵字篩選標題」欄位輸入一個字串，例如 `家庭`、`信心`
   - 列表會只顯示標題內含這個字串的文章
   - 這時可以用：
     - 「**勾選篩選結果**」：把目前顯示的全部設為勾選
     - 「**取消篩選結果**」：把目前顯示的全部取消勾選
   - 這不會動到沒被篩選出的其他項目

3. **用編號範圍選文章**
   - `起` / `迄`，分別輸入數字，例如：
     - 起：`5`
     - 迄：`20`
   - 按「**勾選此範圍**」→ 第 5~20 筆會全部打勾
   - 按「**取消此範圍**」→ 第 5~20 筆的勾選會被取消
   - 這個範圍是針對原始 `links` 陣列的順序（左側編號），不受文字篩選影響

上面這三種方法可以組合使用：
- 先全不選 → 用關鍵字 +「勾選篩選結果」
- 或先全選 → 用編號範圍把不想要的取消

### 第五步：按「2. 開始提取內容」

當你滿意勾選的文章後：

1. 按「**2. 開始提取內容（XX 筆）**」按鈕
2. 工具會：
   - 針對**每一個已勾選的文章網址**，透過 CORS 代理抓 HTML
   - 找文章主體（`article` / `#article` / `.bodyTxt`）
   - 移除各種不必要的區塊（分享按鈕、導航、隱藏元素、自訂 icon 等）
   - 把相對路徑的圖片 / 連結補成 `https://wol.jw.org/...`
   - 產生兩份內容：
     - `content`：已清洗的純文字
     - `htmlContent`：保留排版的 HTML 片段

3. 執行過程中：
   - 狀態欄會顯示「正在提取 (3/20)：某某標題」
   - 進度條會跟著更新
   - 每篇之間會稍微 `setTimeout` 暫停一下，避免被伺服器當成異常流量

完成後，下方會出現一個「提取預覽區」：

- 每篇會顯示：
  - 標題
  - 原始網址
  - 純文字內容的前幾行（預覽）

這裡只是讓你大概確認內容有沒有跑歪，不是給你全部閱讀用。

### 第六步：下載 TXT / HTML

提取完成後，上方右側兩顆下載按鈕就可以用了。

1. **下載 TXT**
   - 檔名：`{fileTitle}.txt`
   - 格式大致如下：

     ```
     ========== WOL_提取內容 ==========
     
     ==========
     標題: 文章 A
     網址: https://wol.jw.org/...
     內容:
     （單行純文字…）
     
     ==========
     標題: 文章 B
     ...
     ```

   - 適合貼進純文字筆記或再進一步處理

2. **下載 HTML（推薦）**
   - 檔名：`{fileTitle}.html`
   - 內容是一個完整、可開啟的單一 HTML 文件：
     - 頂部有大標題（就是你輸入的檔案名稱）
     - 每篇文章一個 card：
       - 標題
       - 來源網址（可點）
       - `htmlContent`（保留圖片、段落、超連結排版）
   - 用瀏覽器打開就可以直接閱讀或列印

## 小提醒

- 這個工具是**個人研究與整理用**的小幫手，不是官方產品。
- 如果遇到：
  - `錯誤: 無法讀取頁面，可能是代理伺服器被阻擋。`
  - 多半是 `corsproxy.io` 暫時掛掉或被限流，可以換時間再試。
- 守望臺型目錄因為會展開兩層，連結多的時候整個流程可能會跑比較久，是正常現象。

---

## 快速總結

- 一般書籍 / 單層目錄：  
  貼那一頁 → 按「獲取頁面連結」→ 馬上看到文章列表 → 勾選 → 「開始提取內容」→ 下載。

- 守望臺兩層目錄：  
  貼「年度 / 卷 / 分類」頁 → 按「獲取頁面連結」→ 工具自動展開子目錄和第三層 → 得到所有文章 → 勾選 → 提取 → 下載。
